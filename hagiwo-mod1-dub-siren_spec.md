# hagiwo-mod1-dub-siren.ino 仕様解析

対象ファイル: `hagiwo-mod1-dub-siren.ino`  
解析日: 2026-06-18

## 1. 概要

このスケッチは、Arduino Nano / ATmega328P を前提に、D11 / F4 から 8bit PWM 音声を出力するダブサイレン風発振器です。

3つのポテンショメータで基本ピッチ、LFO振幅、LFO速度を操作し、ボタンでピッチLFO波形を切り替えます。LFOによって発振周波数を周期的に変調し、サイレン風のピッチ変化を作ります。

主な特徴は次の通りです。

- 出力方式は `tone()` ではなく 8bit PWM。
- 出力ピンは D11 / F4。
- PWMキャリア周波数は Timer2 による約62.5kHz。
- 音声サンプル更新周波数は Timer1 による約15.625kHz。
- 音声波形は256サンプルの8bitサイン波テーブル。
- LFO波形は4種類。
- 初期LFO波形はサイン波。
- SPEED POT が左端付近のときは無音。
- 無音判定にはヒステリシスを使用。
- 発音/無音移行時はPWM出力レベルを20msでフェード。
- POT値には前回平滑化値との平均による簡易平滑化あり。
- 周波数は 130Hz から 3000Hz に制限。
- 通常制御更新は 10ms ごと。
- 実行時の `float`、`sin()`、`map()` は使用しない。

## 2. ハードウェア仕様

| 用途 | Arduino ピン | コード上の定義 |
|---|---:|---|
| 音声PWM出力 | D11 | `PinConfig::SPEAKER` |
| 基本ピッチ POT | A0 | `PinConfig::PITCH_POT` |
| LFO振幅 POT | A1 | `PinConfig::AMP_POT` |
| LFO速度 POT | A2 | `PinConfig::SPEED_POT` |
| 波形切替ボタン | D4 | `PinConfig::BUTTON` |
| LFO状態LED | D3 | `PinConfig::LED` |

ボタン入力は `INPUT_PULLUP` です。そのため、ボタンが押されていないときは HIGH、押されたときは LOW として扱われます。

### 2.1 MOD1回路図上の接続

回路図 `MOD1_回路図.webp` から読み取れる、Arduino Nano ピンと基板上のノブ/ジャック/表示部の対応は次の通りです。

| Arduino ピン | 回路図ネット | 接続先 | 現行スケッチでの用途 |
|---|---|---|---|
| A0 | `A0` | `POT1` / `RV1` のワイパー。`R1 10k` 経由、`C2 0.01uF` でGNDへフィルタ | 基本ピッチ |
| A1 | `A1` | `POT2` / `RV2` のワイパー。`R2 10k` 経由、`C3 0.01uF` でGNDへフィルタ | LFO振幅 |
| A2 | `A2` | `POT3` / `RV3` のワイパー。`R3 10k` 経由、`C4 0.01uF` でGNDへフィルタ | LFO速度 / 無音制御 |
| A3 / D17 | `A3_D17` | `F1` ジャック `J1` のT端子。`R9 1k` 経由、`R12 100k` プルダウン、`D1/D2` クランプ、`C12 0.01uF` | 未使用 |
| A4 | `A4` | `F2` ジャック `J2` のT端子側と同一系統 | 未使用 |
| D9 | `D9` | `F2` ジャック `J2` のT端子。`C6 1uF`、`R4 1k`、`R5 100k` プルダウン、`D3/D6` クランプ、`C5 0.01uF` | 未使用 |
| A5 | `A5` | `F3` ジャック `J3` のT端子側と同一系統 | 未使用 |
| D10 | `D10` | `F3` ジャック `J3` のT端子。`C8 1uF`、`R6 1k`、`R7 100k` プルダウン、`D7/D8` クランプ、`C9 0.01uF` | 未使用 |
| D11 | `D11` | `F4` ジャック `J4` のT端子。`C10 1uF`、`R8 1k`、保護ダイオード経由 | 8bit PWM音声出力 |
| D4 | `D4` | `SW1` 押しボタン。`R11 10k` 経由でGNDへ落ちる | LFO波形切替 |
| D3 | `D3` | LED回路。`R10 10k` とLED `D4` を経由してGND | LFO状態LED |

`POT1`、`POT2`、`POT3` はいずれも `100k` のポテンショメータで、両端が `+5V` と `GND`、ワイパーが各Arduinoアナログ入力へ接続されています。

`F1` から `F4` は `AudioJack2_SwitchT` として描かれており、S端子はGNDに接続されています。T端子が信号線で、各ジャックの信号は抵抗、プルダウン、クランプダイオード、コンデンサを経由してArduinoピンへ入っています。現行スケッチでは `F4` / `D11` のみを音声PWM出力として使用し、`F1`、`F2`、`F3` は使用していません。

### 2.2 ジャックの入出力設定で注意する点

`F1` から `F4` は、回路上はArduinoピンにつながったジャックなので、ソフトウェア上の `pinMode()` 設定次第で入力にも出力にも使えます。ただし、各ジャックの周辺回路と接続ピンの性質が違うため、向きの適性は同じではありません。

| Jack | 主な接続ピン | 入出力の扱い |
|---|---|---|
| F1 | A3 / D17 | アナログ入力またはデジタル入力向き。出力にもできるが、現行回路では入力保護付きのCV/ゲート入力として扱いやすい |
| F2 | A4 と D9 | 入力にも出力にもできるが、A4とD9が同じ信号線につながっているため設定注意 |
| F3 | A5 と D10 | 入力にも出力にもできるが、A5とD10が同じ信号線につながっているため設定注意 |
| F4 | D11 | 現行スケッチでは8bit PWM音声出力。デジタル入力にもできるが、アナログ入力には使えない |

`F2` と `F3` の「設定注意」は、1つのジャック信号線にアナログ入力ピンとデジタルI/Oピンの両方が接続されていることを指します。たとえば `F2` は `A4` と `D9` が同じネット、`F3` は `A5` と `D10` が同じネットです。

外部からジャックへ信号を入れて `analogRead(A4)` / `analogRead(A5)` で読む場合は、対応する `D9` / `D10` を `OUTPUT` にしないようにします。`D9` / `D10` が出力状態でHIGHまたはLOWを駆動していると、外部信号やアナログ入力回路と衝突します。入力用途では、対応するデジタルピンは `INPUT` のままにして高インピーダンスにしておくのが安全です。

逆に `D9` / `D10` からジャックへ出力する場合は、同じネットにある `A4` / `A5` は入力状態のままなら大きな問題にはなりません。ただし、外部機器が同じジャックから信号を駆動している状態でArduino側も出力すると、外部機器とArduino出力がぶつかる可能性があります。

Arduino Nanoでは `A4` と `A5` はI2Cの `SDA` / `SCL` としても使われます。将来I2Cデバイスを追加する場合、`F2` / `F3` のジャック回路とI2C信号が同じピンを共有するため、機能が衝突する可能性があります。

## 3. PWM音声仕様

現行スケッチは `tone()` を使いません。D11はATmega328Pの `OC2A` なので、Timer2をFast PWMに設定して8bit PWM出力します。

| 項目 | 値 |
|---|---:|
| 出力ピン | D11 / OC2A / F4 |
| PWM分解能 | 8bit |
| PWM中心値 | 128 |
| PWMキャリア周波数 | 約62.5kHz |
| 音声サンプル更新周波数 | 約15.625kHz |
| 音声波形 | 256サンプル8bitサイン波テーブル |
| 音量制御 | `audioLevel` 0-255 |
| 周波数制御 | 32bit位相アキュムレータ |

Timer2はD11のPWMキャリア生成に使用します。Timer1はCTCモードで15.625kHzの割り込みを発生させ、割り込み内で次のサンプル値を `OCR2A` へ書き込みます。

音声ISR内では、処理負荷を抑えるために次の処理だけを行います。

1. `audioPhase += audioPhaseIncrement`
2. `audioPhase >> 24` で波形テーブルインデックスを作る
3. `sineTable` から8bitサンプルを読む
4. サンプルを中心値128基準へ変換する
5. `audioLevel` を掛け、`>> 8` でスケールする
6. `OCR2A` へ書く

ISR内では `analogRead()`、`float`、`sin()`、`map()`、割り算、Serial出力は使用しません。

## 4. 操作仕様

### 4.1 PITCH POT

`A0` の値を読み取り、基本周波数 `baseFreq` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 130Hz-2100Hz

変換は整数演算で行い、1023除算の代わりに `>> 10` を使います。

```cpp
baseFreq = 130 + ((pitchSmooth * (2100 - 130)) >> 10)
```

### 4.2 AMP POT

`A1` の値を読み取り、LFOによる周波数変調幅 `lfoDepthHz` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 0Hz-600Hz

この値は音量ではなく、ピッチLFOの変調幅です。PWM音量は通常時 `255` 固定で、発音/無音移行時だけフェード制御されます。

### 4.3 SPEED POT

`A2` の値を読み取り、LFO位相ステップ `lfoStep` に変換します。

- 入力範囲: 0-1023
- 出力範囲: `104`-`8344`

`lfoStep` は16bit LFO位相へ10msごとに加算されます。値が大きいほどLFO周期が速くなります。

また、SPEED POT は無音制御にも使われます。

- `speedValue <= 10`: 無音へ移行
- `speedValue > 20`: 発音へ復帰

10 と 20 の間に幅を持たせることで、境界付近で発音と無音が細かく切り替わるチャタリングを防ぎます。

### 4.4 ボタン

ボタンを押すたびにピッチLFO波形が次の順番で切り替わります。

1. `SINE`
2. `SQUARE`
3. `SAWTOOTH`
4. `REVERSE_SAWTOOTH`
5. `SINE` に戻る

デバウンス時間は 200ms です。

## 5. 周波数生成仕様

通常動作時は10msごとに次の流れで周波数を決めます。

1. POTから `baseFreq`, `lfoDepthHz`, `lfoStep`, `speedValue` を読む。
2. `lfoStep` を `state.lfoPhase` に加算する。
3. 現在のLFO波形に応じて `lfoValue` を整数で計算する。
4. `baseFreq + lfoValue` を `modulatedFreq` とする。
5. `modulatedFreq` を 130Hz-3000Hz に制限する。
6. `modulatedFreq * 274878UL` で32bit音声位相増分を作る。
7. 音声ISRが位相増分に従って8bitサイン波テーブルを読み、D11 PWMへ出力する。
8. `lfoValue > 0` のとき LED を点灯する。

`274878UL` は `2^32 / 15625Hz` の近似値です。15.625kHzサンプル更新を前提に、Hz単位の周波数から位相増分へ変換します。

## 6. LFO波形仕様

LFOは16bit位相 `state.lfoPhase` を使います。上位8bitをテーブル/波形計算のインデックスとして使います。

### 6.1 SINE

`sineTable[index] - 128` を使い、約 `-127` から `+127` の範囲を作ります。

### 6.2 SQUARE

4区間に分けて、固定値とサインテーブル補間を組み合わせます。

- `0x0000 <= phase < 0x4000`: `+127`
- `0x4000 <= phase < 0x8000`: サインテーブルによる正側補間
- `0x8000 <= phase < 0xC000`: `-127`
- `0xC000 <= phase <= 0xFFFF`: サインテーブルによる負側補間

旧実装と同様に、完全に滑らかな波形ではなく区間境界で急変する箇所があります。

### 6.3 SAWTOOTH

上位8bitインデックスから `index - 128` を返します。`-128` から `+127` へ直線的に上がる波形です。

### 6.4 REVERSE_SAWTOOTH

上位8bitインデックスから `127 - index` を返します。`+127` から `-128` へ直線的に下がる波形です。

## 7. 状態管理仕様

状態は `State state` に集約されています。

| メンバー | 役割 |
|---|---|
| `waveformType` | 現在のピッチLFO波形 |
| `lfoPhase` | LFOの現在位相。16bit |
| `lastUpdateTime` | 通常制御更新の前回時刻 |
| `lastButtonPress` | ボタン押下の前回時刻 |
| `isSilent` | 現在無音状態かどうか |
| `sweepActive` | 発音/無音移行中かどうか |
| `sweepStartTime` | 移行開始時刻 |
| `sweepFromLevel` | 移行開始時のPWM出力レベル |
| `sweepToLevel` | 移行終了時のPWM出力レベル |
| `isEnteringSound` | 発音へ入る移行かどうか |

音声ISRとメインループで共有する値は、次の `volatile` 変数です。

| 変数 | 役割 |
|---|---|
| `audioPhase` | 音声波形の32bit位相 |
| `audioPhaseIncrement` | 音声周波数に対応する32bit位相増分 |
| `audioLevel` | PWM出力レベル。0-255 |
| `audioEnabled` | 音声出力有効フラグ |

`audioPhaseIncrement`、`audioLevel`、`audioEnabled` の更新は `noInterrupts()` / `interrupts()` で保護しています。

## 8. 無音・復帰仕様

起動直後は `state.isSilent = true` なので無音状態から始まります。PWM出力は中心値 `128` に固定されます。

発音へ移行する条件:

```cpp
state.isSilent == true
speedValue > 20
sweepActive == false
```

無音へ移行する条件:

```cpp
state.isSilent == false
speedValue <= 10
sweepActive == false
```

移行時間は `SWEEP_DURATION = 20ms` です。PWM版では `audioLevel` を 0-255 の範囲で変化させるため、旧 `tone()` 版と違って実際の出力振幅が変化します。

発音移行:

- `audioEnabled = true`
- `audioLevel` を 0 から 255 へフェード

無音移行:

- `audioLevel` を 255 から 0 へフェード
- 完了後に `audioEnabled = false`
- `OCR2A` を中心値 128 に固定

## 9. 性能・互換性の注意点

### 9.1 Timer使用

PWM版ではTimer1とTimer2を直接設定します。

| Timer | 用途 | 影響 |
|---|---|---|
| Timer1 | 15.625kHzサンプル割り込み | D9/D10のTimer1 PWM機能と衝突する可能性 |
| Timer2 | D11の62.5kHz PWMキャリア | `tone()`、D3/D11の通常PWM用途と衝突する可能性 |
| Timer0 | Arduino標準の `millis()` | 未変更 |

現行スケッチではD9/D10は未使用、D3はLEDの `digitalWrite()` のみなので、Timer1/Timer2の直接使用は現行機能とは衝突しません。

### 9.2 `tone()` との非互換

PWM版では `tone()` / `noTone()` は使用しません。Timer2をD11 PWM出力に直接使うため、`tone()` と併用しない前提です。

### 9.3 整数・固定小数点化

実行時の `float`、`sin()`、`map()` は使っていません。POT変換は整数演算と `>> 10`、LFOと音声波形はテーブル参照、音量スケーリングは乗算と `>> 8` で行います。

### 9.4 音質上の注意

PWM出力はDAC出力ではありません。D11からは約62.5kHzキャリアを含む8bit PWM波形が出ます。F4周辺には `R8 1k`、保護ダイオード、`C10 1uF` がありますが、理想的なライン出力用ローパスフィルタとは限りません。

外部ミキサーやモジュラー機器へ接続する場合は、受け側の入力レベル、PWMキャリア成分、必要に応じた減衰やフィルタを考慮します。

## 10. 実装上の改善候補

### 10.1 音声波形の切替

現行のボタンはピッチLFO波形切替として維持しています。音声波形自体はサイン波固定です。将来、音声波形も矩形波、三角波、ノコギリ波へ切り替えたい場合は、別操作かモード切替が必要です。

### 10.2 LFO SQUARE波形の不連続

SQUARE LFOは旧実装の性格を残すため、固定値とサインテーブル補間を組み合わせています。完全に滑らかではないため、急な周波数変化をさらに抑えたい場合はLFO波形設計を見直す余地があります。

### 10.3 PWM出力フィルタ

よりクリーンな音声にしたい場合は、F4出力後段にPWMキャリアを落とすローパスフィルタや出力レベル調整回路を追加する検討余地があります。
