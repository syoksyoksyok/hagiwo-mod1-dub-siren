# hagiwo-mod1-dub-siren.ino 仕様解析

対象ファイル: `hagiwo-mod1-dub-siren.ino`

バージョン: v1.0.0

ライセンス: MIT License

この仕様書は、HAGIWO / ハギヲさん設計の HAGIWO MOD1 ハードウェア向け非公式ファームウェアの説明です。

## 1. 概要

このスケッチは、Timer1 割り込みで D11 をトグルして、ダブサイレン風の矩形波発振音を出すプログラムです。従来の `tone()` ベース実装は、ソース内のフォールバックとして残しています。

3つのポテンショメータで基本ピッチ、LFO振幅、LFO速度を操作し、ボタンでLFO波形を切り替えます。LFOによって出力周波数を周期的に変調し、サイレン風のピッチ変化を作ります。

主な特徴は次の通りです。

- 出力は Timer1 割り込みによる 5V 系の矩形波。
- 出力ピンは D11。
- LFO波形は4種類。
- 初期波形はサイン波。
- A3 / D17 のGATE入力がHIGHの間だけ発音。
- D9 のKILL入力がHIGHの間は強制ミュート。
- D10 のLFO PAUSE入力がHIGHの間はLFO位相を停止。
- POT3 はLFO速度専用。発音/無音はGATE入力またはボタン長押しで制御。
- ボタン短押しはLFO波形切替、0.23秒以上の長押しは押している間だけ発音。
- LEDはLFO速度で明暗変化し、LFO深さに応じてソフトウェアPWMの最大輝度が変わる。
- POT値には前回平滑化値との平均による簡易平滑化あり。
- 周波数は 130Hz から 3000Hz に制限。
- 通常動作時は 10ms ごとに周波数を更新。

## 2. ハードウェア仕様

| 用途 | Arduino ピン | コード上の定義 |
|---|---:|---|
| 音声出力 | D11 | `PinConfig::SPEAKER` |
| 基本ピッチ POT | A0 | `PinConfig::PITCH_POT` |
| LFO振幅 POT | A1 | `PinConfig::AMP_POT` |
| LFO速度 POT | A2 | `PinConfig::SPEED_POT` |
| GATE入力 | A3 / D17 | `PinConfig::GATE_INPUT` |
| KILL入力 | D9 | `PinConfig::KILL_INPUT` |
| LFO PAUSE入力 | D10 | `PinConfig::LFO_PAUSE_INPUT` |
| 波形切替/手動発音ボタン | D4 | `PinConfig::BUTTON` |
| LFO表示LED | D3 | `PinConfig::LED` |

ボタン入力は `INPUT_PULLUP` です。そのため、ボタンが押されていないときは HIGH、押されたときは LOW として扱われます。

### 2.1 MOD1回路図上の接続

HAGIWO MOD1 の回路図から読み取れる、Arduino Nano ピンと基板上のノブ/ジャック/表示部の対応は次の通りです。

| Arduino ピン | 回路図ネット | 接続先 | 現行スケッチでの用途 |
|---|---|---|---|
| A0 | `A0` | `POT1` / `RV1` のワイパー。`R1 10k` 経由、`C2 0.01uF` でGNDへフィルタ | 基本ピッチ `PinConfig::PITCH_POT` |
| A1 | `A1` | `POT2` / `RV2` のワイパー。`R2 10k` 経由、`C3 0.01uF` でGNDへフィルタ | LFO振幅 `PinConfig::AMP_POT` |
| A2 | `A2` | `POT3` / `RV3` のワイパー。`R3 10k` 経由、`C4 0.01uF` でGNDへフィルタ | LFO速度 `PinConfig::SPEED_POT` |
| A3 / D17 | `A3_D17` | `F1` ジャック `J1` のT端子。`R9 1k` 経由、`R12 100k` プルダウン、`D1/D2` クランプ、`C12 0.01uF` | GATE入力 `PinConfig::GATE_INPUT` |
| A4 | `A4` | `F2` ジャック `J2` のT端子側と同一系統 | D9と同一ネットのため直接は未使用 |
| D9 | `D9` | `F2` ジャック `J2` のT端子。`C6 1uF`、`R4 1k`、`R5 100k` プルダウン、`D3/D6` クランプ、`C5 0.01uF` | KILL入力 `PinConfig::KILL_INPUT` |
| A5 | `A5` | `F3` ジャック `J3` のT端子側と同一系統 | D10と同一ネットのため直接は未使用 |
| D10 | `D10` | `F3` ジャック `J3` のT端子。`C8 1uF`、`R6 1k`、`R7 100k` プルダウン、`D7/D8` クランプ、`C9 0.01uF` | LFO PAUSE入力 `PinConfig::LFO_PAUSE_INPUT` |
| D11 | `D11` | `F4` ジャック `J4` のT端子。`C10 1uF`、`R8 1k`、保護ダイオード経由 | 音声出力 `PinConfig::SPEAKER` |
| D4 | `D4` | `SW1` 押しボタン。`R11 10k` 経由でGNDへ落ちる | 短押しで波形切替、長押しで手動発音 `PinConfig::BUTTON` |
| D3 | `D3` | LED回路。`R10 10k` とLED `D4` を経由してGND | LFO表示LED `PinConfig::LED` |

`POT1`、`POT2`、`POT3` はいずれも `100k` のポテンショメータで、両端が `+5V` と `GND`、ワイパーが各Arduinoアナログ入力へ接続されています。

`F1` から `F4` は `AudioJack2_SwitchT` として描かれており、S端子はGNDに接続されています。T端子が信号線で、各ジャックの信号は抵抗、プルダウン、クランプダイオード、コンデンサを経由してArduinoピンへ入っています。現行スケッチでは `F4` / `D11` を音声出力、`F1` / `A3_D17` をGATE入力、`F2` / `D9` をKILL入力、`F3` / `D10` をLFO PAUSE入力として使用しています。

### 2.2 ジャックの入出力設定で注意する点

`F1` から `F4` は、回路上はArduinoピンにつながったジャックなので、ソフトウェア上の `pinMode()` 設定次第で入力にも出力にも使えます。ただし、各ジャックの周辺回路と接続ピンの性質が違うため、向きの適性は同じではありません。

| Jack | 主な接続ピン | 入出力の扱い |
|---|---|---|
| F1 | A3 / D17 | 現行スケッチではGATE入力。外部信号は基板上のプルダウンとクランプを通って入る |
| F2 | A4 と D9 | 現行スケッチではD9をKILL入力として使用。A4とD9が同じ信号線につながっているため設定注意 |
| F3 | A5 と D10 | 現行スケッチではD10をLFO PAUSE入力として使用。A5とD10が同じ信号線につながっているため設定注意 |
| F4 | D11 | 現行スケッチでは音声出力。デジタル入力にもできるが、アナログ入力には使えない |

`F2` と `F3` の「設定注意」は、1つのジャック信号線にアナログ入力ピンとデジタルI/Oピンの両方が接続されていることを指します。たとえば `F2` は `A4` と `D9` が同じネット、`F3` は `A5` と `D10` が同じネットです。

このため、外部からジャックへ信号を入れて `analogRead(A4)` / `analogRead(A5)` で読む場合は、対応する `D9` / `D10` を `OUTPUT` にしないようにします。`D9` / `D10` が出力状態でHIGHまたはLOWを駆動していると、外部信号やアナログ入力回路と衝突します。入力用途では、対応するデジタルピンは `INPUT` のままにして高インピーダンスにしておくのが安全です。`INPUT_PULLUP` も信号にバイアスを加えるため、意図がある場合だけ使います。

逆に `D9` / `D10` からジャックへ出力する場合は、同じネットにある `A4` / `A5` は入力状態のままなら大きな問題にはなりません。ただし、外部機器が同じジャックから信号を駆動している状態でArduino側も出力すると、外部機器とArduino出力がぶつかる可能性があります。出力ジャックとして使う場合は、外部から同時に信号を入れない前提にします。

また、Arduino Nanoでは `A4` と `A5` はI2Cの `SDA` / `SCL` としても使われます。将来I2Cデバイスを追加する場合、`F2` / `F3` のジャック回路とI2C信号が同じピンを共有するため、機能が衝突する可能性があります。

`D11` が音声出力向きなのは、現行スケッチが D11 をデジタル出力としてHIGH/LOWに切り替えて矩形波を作るためです。アナログ入力ピンではなくデジタル出力ピンに接続されたジャックが適しています。`F4` は `D11` の単独ネットとして扱われ、`F2` / `F3` のように別のアナログ入力ピンと共有されていないため、音声出力専用として使いやすい構成です。

ただし、この出力はDAC音声ではなく5V系の矩形波です。音量制御もできません。`F4` 周辺には `R8 1k`、保護ダイオード、`C10 1uF` があり、ジャック接続時の保護や高周波ノイズ低減に寄与しますが、一般的なラインレベル音声出力そのものではありません。外部ミキサーやモジュラー機器へ接続する場合は、受け側の入力レベルと必要に応じた減衰を考慮します。

## 3. 操作仕様

### 3.1 PITCH POT

`A0` の値を読み取り、基本周波数 `baseFreq` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 130Hz-2100Hz

変換式は Arduino の `map()` を使っています。

```cpp
map(pitchValue, 0, 1023, 130, 2100)
```

### 3.2 AMP POT

`A1` の値を読み取り、LFOによる周波数変調幅 `amplitude` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 0-600

単位は直接Hzとして扱われ、最終的に `baseFreq + lfoValue` の形で周波数に加算されます。この値はLEDの最大輝度にも使われ、深さが大きいほどLEDの明暗変化が大きくなります。

### 3.3 SPEED POT

`A2` の値を読み取り、LFOの進行量 `step` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 0.01-0.8

この `step` は更新ごとに `state.angle` へ加算されます。更新周期は 10ms なので、値が大きいほどLFO周期が速くなります。

現在の実装では、POT3はLFO速度だけを担当します。発音/無音はA3 / D17 のGATE入力、D4ボタン長押し、D9 KILL入力で制御します。D10 LFO PAUSE入力がHIGHの間は、この速度設定に関係なくLFO位相が停止します。

### 3.4 GATE入力

`A3 / D17` をデジタル入力として読みます。GATE入力がHIGHの間だけD11へ発音し、LOWの間は音声エンジンを停止します。LOWからHIGHへ変化したときは `state.angle` を `0.0` に戻し、LFO位相を先頭から再開します。

F1回路には `R12 100k` のプルダウンがあるため、コード側は `INPUT` として設定し、`INPUT_PULLUP` は使いません。

### 3.5 ボタン

D4ボタンは押している時間で動作が変わります。

- 0.23秒未満の短押し: LFO波形を次へ切り替える。
- 0.23秒以上の長押し: 押している間だけ手動GATEとして発音する。

短押し時のLFO波形切替順は次の通りです。

1. `SINE`
2. `SQUARE`
3. `SAWTOOTH`
4. `REVERSE_SAWTOOTH`
5. `SINE` に戻る

短押し判定はボタンを離したタイミングで行います。長押しで発音した場合は、ボタンを離しても波形切替は行いません。

### 3.6 KILL入力

`D9` をデジタル入力として読みます。D9がHIGHの間は、A3 GATEやD4ボタン長押しが有効でも音声エンジンを停止して強制ミュートします。KILL中もLFOとLED表示は動き続けます。

### 3.7 LFO PAUSE入力

`D10` をデジタル入力として読みます。D10がHIGHの間は `state.angle` を進めず、LFOを現在位置で停止します。音声ピッチとLED輝度は停止中のLFO位置を保持します。

## 4. 周波数生成仕様

通常動作時は次の流れで周波数を決めます。

1. POTから `baseFreq`, `amplitude`, `step` を読む。
2. D9 KILL入力がHIGHなら強制的に非発音状態にする。
3. KILLがLOWで、A3 / D17 のGATE入力またはD4ボタン長押しが有効なら発音状態、どちらも無効なら音声エンジンを停止する。
4. 非発音状態から発音状態へ変化したら `state.angle` を `0.0` に戻す。
5. D10 LFO PAUSE入力がLOWなら `step` を `state.angle` に加算し、HIGHなら加算しない。
6. `state.angle` が `TWO_PI` 以上なら `TWO_PI` を引いて循環させる。
7. 現在の波形に応じて `lfoValue` を計算する。
8. `baseFreq + lfoValue` を `modulatedFreq` とする。
9. `modulatedFreq` を 130Hz-3000Hz に制限する。
10. `setAudioFrequency(modulatedFreq)` でD11の矩形波周波数を更新する。
11. `lfoValue` を0-最大輝度へ連続変換し、LEDのPWM輝度へ反映する。最大輝度はLFO深さに応じる。

## 5. LFO波形仕様

### 5.1 SINE

```cpp
sin(state.angle) * amplitude
```

サイン波で、`-amplitude` から `+amplitude` の範囲を滑らかに変化します。

### 5.2 SQUARE

コード上は「Smoothed square wave」とコメントされていますが、実装は完全な矩形波ではなく、区間ごとに固定値とサインカーブを組み合わせています。

- `0 <= angle < PI/2`: `+amplitude`
- `PI/2 <= angle < PI`: `amplitude * sin(...)`
- `PI <= angle < 1.5PI`: `-amplitude`
- `1.5PI <= angle < 2PI`: `-amplitude * sin(...)`

注意点として、区間境界で値が急に変わる箇所があります。完全に滑らかな波形ではありません。

### 5.3 SAWTOOTH

```cpp
((state.angle / TWO_PI) * 2 - 1) * amplitude
```

`-amplitude` から `+amplitude` へ直線的に上がる波形です。

### 5.4 REVERSE_SAWTOOTH

```cpp
(1.0 - (state.angle / TWO_PI)) * 2 * amplitude - amplitude
```

`+amplitude` から `-amplitude` へ直線的に下がる波形です。

## 6. 状態管理仕様

状態は `State state` に集約されています。

| メンバー | 役割 |
|---|---|
| `waveformType` | 現在のLFO波形 |
| `angle` | LFOの現在位相 |
| `lastUpdateTime` | 通常動作更新の前回時刻 |
| `buttonPressStart` | ボタンを押し始めた時刻 |
| `buttonWasPressed` | 前回ループ時にボタンが押されていたか |
| `buttonLongPressActive` | ボタン長押しによる手動発音が有効か |
| `lastAudioActive` | 前回ループ時に発音状態だったか。立ち上がり検出に使う |
| `ledBrightness` | LFO値とLFO深さから計算したソフトウェアPWM用LED輝度。0-255 |

## 7. GATE入力と発音仕様

現在の実装では、POT3による無音制御はありません。発音/無音はA3 / D17 のGATE入力、D4ボタン長押し、D9 KILL入力で制御します。

- D9 KILL HIGH: GATEやボタン状態に関係なく音声エンジンを停止して強制ミュートする。
- D9 KILL LOW かつ、GATE HIGH またはボタン長押し中: `updateNormalOperation()` を呼び、D11へ矩形波を出力する。
- D9 KILL LOW かつ、GATE LOW かつボタン長押しなし: 音声エンジンを停止し、D11をLOWにする。LEDは発音状態に関係なくLFO表示を続ける。

非発音状態から発音状態へ変化したときは、`state.angle = 0.0` と `state.lastUpdateTime = 0` を設定します。これにより、GATEまたはボタン長押しで発音が始まるタイミングでLFOが先頭から再開します。D10 LFO PAUSEがHIGHの間は、発音中でもLFO位相を進めません。

## 8. 実装メモ

POT読み取りは前回平滑化値との2サンプル平均です。初回の平滑化値は0なので、起動直後の1回目だけ実値より低めになります。

`A3 / D17` は基板側にプルダウンがある前提で `INPUT` にしています。`INPUT_PULLUP` は外部GATE信号にバイアスを加えるため使っていません。

現在の音声エンジンはD11の矩形波周波数だけを制御するため、このスケッチでは音量フェードや実振幅制御は行っていません。
D3のハードウェアPWMはTimer2系で、ソース内に残している `tone()` フォールバックと衝突する可能性があります。そのためLEDの明るさ制御は `analogWrite()` ではなく、`micros()` を使ったソフトウェアPWMで行います。LED輝度はLFO値を0-最大輝度へ連続変換するため、A2を左端にしてLFOが遅い場合もゆっくりした明暗変化として確認できます。

## 9. コンパイル互換性の注意点

### 9.1 C++17 構造化束縛

以前の版ではC++17の構造化束縛を使っていましたが、現在はArduino AVR環境との互換性を優先して通常の構造体受け取りにしています。

```cpp
PotValues pots = readPotValues();
```

### 9.2 static constexpr float

`AudioConfig` 内の `STEP_MIN` と `STEP_MAX` は `static constexpr float` です。

```cpp
static constexpr float STEP_MIN = 0.01;
static constexpr float STEP_MAX = 0.8;
```

Arduino AVR系ではクラス内の非整数定数は `constexpr` にしておく方が安全です。

### 9.3 map() の戻り値

Arduino の `map()` は整数演算を行い、戻り値は `long` です。

このコードでは `STEP_MIN * 1000` と `STEP_MAX * 1000` を使って 10-800 に変換してから 1000.0 で割ることで、小数のステップ値を作っています。

## 10. 実装上の注意点・改善候補

### 10.1 矩形波出力は音量フェードできない

現在の実装では音量スイープ処理を入れていません。D11の矩形波出力は周波数のみを制御しており、実振幅制御を持たないためです。

本当にポップノイズ低減のためのフェードを行うなら、PWM出力、外部VCA、DAC、またはタイマ割り込みによる波形生成など別方式が必要です。

### 10.2 SQUARE波形は境界で不連続

SQUARE波形は「Smoothed」とコメントされていますが、区間境界で急変する箇所があります。

例えば `angle` が `PI/2` に到達した直後、戻り値が `+amplitude` から `0` へ急変します。急な周波数変化を避けたい場合は、波形設計を見直す余地があります。

### 10.3 初回POT読み取りが半分になる

平滑化用の前回値が 0 で初期化されているため、初回読み取りは実値の約半分になります。起動直後の反応を正確にしたい場合は、初回だけ実読み取り値をそのまま使う処理が考えられます。
