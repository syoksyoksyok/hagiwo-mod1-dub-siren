# hagiwo-mod1-dub-siren.ino Technical Specification

Target file: `hagiwo-mod1-dub-siren.ino`

Version: v1.0.0

License: MIT License

This document describes an unofficial firmware for the HAGIWO MOD1 hardware designed by HAGIWO.

## 1. Overview

This sketch uses a Timer1 interrupt to toggle D11 and generate a dub-siren-style square-wave oscillator sound. The older `tone()`-based implementation remains in the source as a fallback.

The three potentiometers control base pitch, LFO modulation depth, and LFO speed. The button changes the LFO waveform. The LFO periodically modulates the output frequency to create siren-style pitch movement.

Main features:

- 5V square-wave output generated by a Timer1 interrupt.
- Output pin is D11.
- Four LFO waveforms.
- Initial waveform is sine.
- The A3 / D17 GATE input enables sound while HIGH.
- The D9 KILL input force-mutes sound while HIGH.
- The D10 LFO PAUSE input stops the LFO phase while HIGH.
- POT3 is dedicated to LFO speed. Sound on/off is controlled by the GATE input or button hold.
- A short button press changes the LFO waveform. Holding the button for 0.23 seconds or longer enables sound while held.
- The LED follows the LFO speed, and its maximum software-PWM brightness depends on the LFO depth.
- Potentiometer values use simple smoothing based on the previous smoothed value.
- Frequency is limited to 130Hz through 3000Hz.
- During normal operation, the output frequency is updated every 10ms.

## 2. Hardware Specification

| Purpose | Arduino pin | Code definition |
|---|---:|---|
| Audio output | D11 | `PinConfig::SPEAKER` |
| Base pitch pot | A0 | `PinConfig::PITCH_POT` |
| LFO depth pot | A1 | `PinConfig::AMP_POT` |
| LFO speed pot | A2 | `PinConfig::SPEED_POT` |
| GATE input | A3 / D17 | `PinConfig::GATE_INPUT` |
| KILL input | D9 | `PinConfig::KILL_INPUT` |
| LFO PAUSE input | D10 | `PinConfig::LFO_PAUSE_INPUT` |
| Waveform select / manual gate button | D4 | `PinConfig::BUTTON` |
| LFO indicator LED | D3 | `PinConfig::LED` |

The button input uses `INPUT_PULLUP`. Therefore, the button reads HIGH when released and LOW when pressed.

The following table maps the Arduino Nano pins to the knobs, jacks, and indicator on the HAGIWO MOD1 schematic.

| Arduino pin | Schematic net | Connection | Use in this sketch |

| A1 | `A1` | Wiper of `POT2` / `RV2`, through `R2 10k`, filtered to GND by `C3 0.01uF` | LFO depth `PinConfig::AMP_POT` |

| A3 / D17 | `A3_D17` | T terminal of the `F1` jack `J1`, through `R9 1k`, with `R12 100k` pulldown, `D1/D2` clamps, and `C12 0.01uF` | GATE input `PinConfig::GATE_INPUT` |
| A4 | `A4` | Same signal path as the T terminal of the `F2` jack `J2` | Not used directly because it shares the same net as D9 |
| D9 | `D9` | T terminal of the `F2` jack `J2`, with `C6 1uF`, `R4 1k`, `R5 100k` pulldown, `D3/D6` clamps, and `C5 0.01uF` | KILL input `PinConfig::KILL_INPUT` |
| A5 | `A5` | Same signal path as the T terminal of the `F3` jack `J3` | Not used directly because it shares the same net as D10 |

`POT1`, `POT2`, and `POT3` are all `100k` potentiometers. Their ends are connected to `+5V` and `GND`, and each wiper connects to an Arduino analog input.

`F1` through `F4` are shown as `AudioJack2_SwitchT` parts. The S terminal is connected to GND. The T terminal is the signal line, and each jack signal reaches an Arduino pin through resistors, pulldowns, clamp diodes, and capacitors. In this sketch, `F4` / `D11` is used as audio output, `F1` / `A3_D17` as GATE input, `F2` / `D9` as KILL input, and `F3` / `D10` as LFO PAUSE input.

### 2.2 Notes on Jack Input and Output Settings

`F1` through `F4` are jacks connected to Arduino pins, so the software `pinMode()` setting can make them inputs or outputs. However, each jack has different surrounding circuitry and pin characteristics, so they are not equally suitable for every direction.

| Jack | Main connected pins | Input/output notes |
|---|---|---|
| F1 | A3 / D17 | Used as GATE input in this sketch. External signals pass through the board pulldown and clamps. |
| F2 | A4 and D9 | Used as KILL input via D9 in this sketch. A4 and D9 share the same signal line, so pin direction must be handled carefully. |
| F3 | A5 and D10 | Used as LFO PAUSE input via D10 in this sketch. A5 and D10 share the same signal line, so pin direction must be handled carefully. |
| F4 | D11 | Used as audio output in this sketch. It can also be a digital input, but it cannot be used as an analog input. |

The important point for `F2` and `F3` is that one jack signal line is connected to both an analog input pin and a digital I/O pin. For example, `F2` shares a net between `A4` and `D9`, and `F3` shares a net between `A5` and `D10`.

For this reason, when reading an external jack signal with `analogRead(A4)` or `analogRead(A5)`, do not configure the corresponding `D9` or `D10` pin as `OUTPUT`. If `D9` or `D10` drives HIGH or LOW while configured as output, it can fight the external signal or the analog input circuit. For input use, keep the corresponding digital pin as `INPUT` so it remains high impedance. Use `INPUT_PULLUP` only when that bias is intentional.

Conversely, when outputting from `D9` or `D10` to the jack, the shared `A4` or `A5` pin is usually not a problem if it remains an input. However, if external equipment is also driving the same jack, the external device and the Arduino output can conflict. When using these jacks as outputs, assume that no external source is driving the same jack at the same time.

On the Arduino Nano, `A4` and `A5` are also the I2C `SDA` and `SCL` pins. If an I2C device is added in the future, the `F2` / `F3` jack circuits and the I2C signals will share the same pins, which can create a functional conflict.

`D11` is suitable for audio output because this sketch drives D11 HIGH and LOW as a digital output to generate a square wave. A jack connected to a digital output pin is appropriate for this purpose. `F4` is treated as a dedicated `D11` net and is not shared with another analog input pin like `F2` and `F3`, so it is convenient as a dedicated audio output.

However, this output is not DAC audio; it is a 5V square wave. There is no output volume control. Around `F4`, `R8 1k`, protection diodes, and `C10 1uF` help with protection and high-frequency noise reduction when connecting a jack, but the output is not a normal line-level audio output. When connecting it to an external mixer or modular equipment, consider the receiving input level and use attenuation if needed.

## 3. Control Specification

### 3.1 PITCH POT

The `A0` value is read and converted to the base frequency `baseFreq`.

- Input range: 0-1023
- Output range: 130Hz-2100Hz

The conversion uses Arduino `map()`.

```cpp
map(pitchValue, 0, 1023, 130, 2100)
```

### 3.2 AMP POT

The `A1` value is read and converted to the LFO frequency-modulation width `amplitude`.

- Input range: 0-1023
- Output range: 0-600

The value is treated directly as Hz and is eventually added to the frequency as `baseFreq + lfoValue`. This value is also used for the LED maximum brightness, so deeper LFO settings create deeper LED brightness movement.

### 3.3 SPEED POT

The `A2` value is read and converted to the LFO phase increment `step`.

- Input range: 0-1023
- Output range: 0.01-0.8

This `step` value is added to `state.angle` on each update. Because the update interval is 10ms, larger values create a faster LFO cycle.

In the current implementation, POT3 controls only LFO speed. Sound on/off is controlled by the A3 / D17 GATE input, the D4 button hold, and the D9 KILL input. While the D10 LFO PAUSE input is HIGH, the LFO phase stops regardless of this speed setting.

### 3.4 GATE Input

`A3 / D17` is read as a digital input. Sound is output on D11 only while the GATE input is HIGH. When it is LOW, the audio engine is stopped. When sound changes from inactive to active, `state.angle` is not reset, so the LFO phase continues from its current position. The same applies when sound starts from the D4 button hold. See "7. GATE Input and Sound Output Behavior" for details.

Because the F1 circuit has an `R12 100k` pulldown, the code configures the pin as `INPUT` and does not use `INPUT_PULLUP`.

### 3.5 Button

The D4 button behavior depends on how long the button is held.

- Short press under 0.23 seconds: changes to the next LFO waveform.
- Hold for 0.23 seconds or longer: acts as a manual gate and outputs sound while held.

The LFO waveform changes in this order:

1. `SINE`
2. `SQUARE`
3. `SAWTOOTH`
4. `REVERSE_SAWTOOTH`
5. Back to `SINE`

Short-press detection happens when the button is released. If the button was held long enough to produce sound, releasing it does not change the waveform.

### 3.6 KILL Input

`D9` is read as a digital input. While D9 is HIGH, the audio engine is stopped and force-muted even if the A3 GATE or D4 button hold is active. The LFO and LED indicator continue moving during KILL.

### 3.7 LFO PAUSE Input

`D10` is read as a digital input. While D10 is HIGH, `state.angle` does not advance and the LFO stays at its current position. The audio pitch and LED brightness keep the LFO value from the paused position.

## 4. Frequency Generation

During normal operation, the frequency is determined as follows.

1. Read `baseFreq`, `amplitude`, and `step` from the potentiometers.
2. If the D9 KILL input is HIGH, force the audio state inactive.
3. If KILL is LOW and either the A3 / D17 GATE input or D4 button hold is active, audio is active. If both are inactive, stop the audio engine.
4. When audio changes from inactive to active, set `state.angle` to `0.0`.
5. If the D10 LFO PAUSE input is LOW, add `step` to `state.angle`. If it is HIGH, do not add it.
6. If `state.angle` is `TWO_PI` or greater, subtract `TWO_PI` to wrap the phase.
7. Calculate `lfoValue` according to the current waveform.
8. Set `modulatedFreq` to `baseFreq + lfoValue`.
9. Clamp `modulatedFreq` to 130Hz-3000Hz.
10. Update the D11 square-wave frequency with `setAudioFrequency(modulatedFreq)`.
11. Convert `lfoValue` continuously to a 0-to-maximum brightness value and apply it to the LED PWM brightness. The maximum brightness depends on the LFO depth.

## 5. LFO Waveforms

### 5.1 SINE

```cpp
sin(state.angle) * amplitude
```

The sine waveform changes smoothly from `-amplitude` to `+amplitude`.

### 5.2 SQUARE

The square waveform switches directly between positive and negative modulation values once per LFO cycle.

```cpp
state.angle < PI ? amplitude : -amplitude
```

- `0 <= angle < PI`: `+amplitude`
- `PI <= angle < 2PI`: `-amplitude`

This is an intentionally hard-edged LFO waveform. It creates one high section and one low section per LFO cycle.

### 5.3 SAWTOOTH

```cpp
((state.angle / TWO_PI) * 2 - 1) * amplitude
```

This waveform rises linearly from `-amplitude` to `+amplitude`.

### 5.4 REVERSE SAWTOOTH

```cpp
(1.0 - (state.angle / TWO_PI)) * 2 * amplitude - amplitude
```

This waveform falls linearly from `+amplitude` to `-amplitude`.

## 6. State Management

State is collected in `State state`.

| Member | Role |
|---|---|
| `waveformType` | Current LFO waveform |
| `angle` | Current LFO phase |
| `lastUpdateTime` | Previous normal-operation update time |
| `buttonPressStart` | Time when the button was pressed |
| `buttonWasPressed` | Whether the button was pressed in the previous loop |
| `buttonLongPressActive` | Whether manual sound output from a button hold is active |
| `lastAudioActive` | Whether audio was active in the previous loop. Used for rising-edge detection |
| `ledBrightness` | Software-PWM LED brightness calculated from the LFO value and LFO depth. Range is 0-255 |

## 7. GATE Input and Sound Output Behavior

In the current implementation, POT3 does not control silence. Sound on/off is controlled by the A3 / D17 GATE input, D4 button hold, and D9 KILL input.

- D9 KILL HIGH: stop the audio engine and force-mute regardless of the GATE or button state.
- D9 KILL LOW and either GATE HIGH or button hold active: call `updateNormalOperation()` and output a square wave on D11.
- D9 KILL LOW, GATE LOW, and no button hold: stop the audio engine and drive D11 LOW. The LED continues showing the LFO regardless of the audio state.

When audio changes from inactive to active, the sketch does not change `state.angle`. The LFO continues from its current position when sound starts by GATE or button hold. On sound start, the current LFO value is used to set the output frequency immediately, and only the start fade is applied. While D10 LFO PAUSE is HIGH, the LFO phase does not advance even during sound output.

## 8. Implementation Notes

Potentiometer readings use a 2-sample average with the previous smoothed value. Because the initial smoothed value is 0, the first reading after startup can be lower than the actual value.

`A3 / D17` is configured as `INPUT` because the board is expected to provide a pulldown. `INPUT_PULLUP` is not used because it would bias the external GATE signal.

The current audio engine controls only the D11 square-wave frequency, so this sketch does not implement true amplitude control. To reduce output-stop clicks, it uses a short stop-only pseudo fade-out by reducing the output pulse density before driving D11 LOW.

D3 hardware PWM uses Timer2 on the ATmega328P, which can conflict with the `tone()` fallback that remains in the source. For this reason, LED brightness is controlled by software PWM using `micros()` instead of `analogWrite()`. LED brightness is continuously converted from the LFO value into a 0-to-maximum brightness range, so slow LED movement is still visible when A2 is near the far-left position and the LFO is very slow.

## 9. Compile Compatibility Notes

### 9.1 C++17 Structured Bindings

Earlier versions used C++17 structured bindings. The current code uses normal struct assignment for compatibility with the Arduino AVR environment.

```cpp
PotValues pots = readPotValues();
```

### 9.2 static constexpr float

`STEP_MIN` and `STEP_MAX` in `AudioConfig` are `static constexpr float`.

```cpp
static constexpr float STEP_MIN = 0.01;
static constexpr float STEP_MAX = 0.8;
```

On Arduino AVR targets, class-scope non-integer constants are safer when declared as `constexpr`.

### 9.3 map() Return Value

Arduino `map()` uses integer arithmetic and returns `long`.

This code creates a fractional step value by mapping `STEP_MIN * 1000` and `STEP_MAX * 1000` to 10-800, then dividing by 1000.0.

## 10. Implementation Notes and Possible Improvements

### 10.1 Square-Wave Output Has No True Amplitude Control

The D11 square-wave output controls only frequency and has no true amplitude control. The current implementation includes a short stop-only pseudo fade-out to reduce output-stop clicks, but it is not a true volume envelope.

A real fade or voltage-controlled amplitude would require a different method, such as PWM output, an external VCA, a DAC, or timer-interrupt-based waveform generation.

### 10.2 SQUARE Waveform Has Intentional Hard Edges

The SQUARE LFO waveform switches directly between `+amplitude` and `-amplitude` at the half-cycle boundary. This is intentional and gives the waveform its square behavior.

If smoother pitch movement is desired, use `SINE`, `SAWTOOTH`, or `REVERSE_SAWTOOTH`, or revise the SQUARE waveform to add a short transition region.

### 10.3 First Potentiometer Reading Can Be Low

The previous values used for smoothing are initialized to 0, so the first reading after startup can be about half of the actual value. During normal operation, the values work as a 2-sample average with the previous smoothed value.
