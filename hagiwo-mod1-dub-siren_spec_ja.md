# hagiwo-mod1-dub-siren.ino 技術仕様

対象ファイル: `hagiwo-mod1-dub-siren.ino`

バージョン: v1.0.0

ライセンス: MIT License

このドキュメントは、HAGIWO さん設計の HAGIWO MOD1 ハードウェア向け非公式ファームウェアについて説明します。

## 1. 概要

このスケッチは、Timer1 割り込みで D11 をトグルし、ダブサイレン風の矩形波発振音を生成します。以前の `tone()` ベース実装は、ソース内にフォールバックとして残しています。

3つのポテンショメータで、基本ピッチ、LFO 変調深さ、LFO 速度を操作します。ボタンで LFO 波形を変更します。LFO は出力周波数を周期的に変調し、サイレン風のピッチ変化を作ります。

主な機能:

- Timer1 割り込みによる 5V 矩形波出力。
- 出力ピンは D11。
- LFO 波形は4種類。
- 初期波形はサイン波。
- A3 / D17 GATE 入力が HIGH の間だけ発音。
- D9 KILL 入力が HIGH の間は強制ミュート。
- D10 LFO PAUSE 入力が HIGH の間は LFO 位相を停止。
- POT3 は LFO 速度専用。発音のオン/オフは GATE 入力またはボタン長押しで制御。
- ボタン短押しで LFO 波形を変更。0.23 秒以上の長押しで、押している間だけ発音。
- LED は LFO 速度に追従し、ソフトウェア PWM の最大輝度は LFO 深さに応じて変化。
- ポテンショメータ値は、前回の平滑化値を使った簡易平滑化を行う。
- 周波数は 130Hz から 3000Hz に制限。
- 通常動作時は 10ms ごとに出力周波数を更新。

## 2. ハードウェア仕様

| 用途 | Arduino ピン | コード上の定義 |
|---|---:|---|
| オーディオ出力 | D11 | `PinConfig::SPEAKER` |
| 基本ピッチ POT | A0 | `PinConfig::PITCH_POT` |
| LFO 深さ POT | A1 | `PinConfig::AMP_POT` |
| LFO 速度 POT | A2 | `PinConfig::SPEED_POT` |
| GATE 入力 | A3 / D17 | `PinConfig::GATE_INPUT` |
| KILL 入力 | D9 | `PinConfig::KILL_INPUT` |
| LFO PAUSE 入力 | D10 | `PinConfig::LFO_PAUSE_INPUT` |
| 波形選択 / 手動ゲートボタン | D4 | `PinConfig::BUTTON` |
| LFO インジケータ LED | D3 | `PinConfig::LED` |

ボタン入力は `INPUT_PULLUP` を使用します。そのため、ボタンを離しているときは HIGH、押しているときは LOW として読み取られます。

### 2.1 MOD1 回路図上の接続

次の表は、HAGIWO MOD1 回路図上の Arduino Nano ピンと、ノブ、ジャック、インジケータの対応です。

| Arduino ピン | 回路図ネット | 接続 | このスケッチでの用途 |
|---|---|---|---|
| A0 | `A0` | `POT1` / `RV1` のワイパー。`R1 10k` 経由、`C2 0.01uF` で GND へフィルタ | 基本ピッチ `PinConfig::PITCH_POT` |
| A1 | `A1` | `POT2` / `RV2` のワイパー。`R2 10k` 経由、`C3 0.01uF` で GND へフィルタ | LFO 深さ `PinConfig::AMP_POT` |
| A2 | `A2` | `POT3` / `RV3` のワイパー。`R3 10k` 経由、`C4 0.01uF` で GND へフィルタ | LFO 速度 `PinConfig::SPEED_POT` |
| A3 / D17 | `A3_D17` | `F1` ジャック `J1` の T 端子。`R9 1k` 経由、`R12 100k` プルダウン、`D1/D2` クランプ、`C12 0.01uF` | GATE 入力 `PinConfig::GATE_INPUT` |
| A4 | `A4` | `F2` ジャック `J2` の T 端子と同じ信号経路 | D9 と同じネットを共有するため直接は未使用 |
| D9 | `D9` | `F2` ジャック `J2` の T 端子。`C6 1uF`、`R4 1k`、`R5 100k` プルダウン、`D3/D6` クランプ、`C5 0.01uF` | KILL 入力 `PinConfig::KILL_INPUT` |
| A5 | `A5` | `F3` ジャック `J3` の T 端子と同じ信号経路 | D10 と同じネットを共有するため直接は未使用 |
| D10 | `D10` | `F3` ジャック `J3` の T 端子。`C8 1uF`、`R6 1k`、`R7 100k` プルダウン、`D7/D8` クランプ、`C9 0.01uF` | LFO PAUSE 入力 `PinConfig::LFO_PAUSE_INPUT` |
| D11 | `D11` | `F4` ジャック `J4` の T 端子。`C10 1uF`、`R8 1k`、保護ダイオード経由 | オーディオ出力 `PinConfig::SPEAKER` |
| D4 | `D4` | プッシュボタン `SW1`。`R11 10k` 経由で GND へ接続 | 短押しで波形変更、長押しで手動発音 `PinConfig::BUTTON` |
| D3 | `D3` | `R10 10k` と LED `D4` を経由して GND へ接続される LED 回路 | LFO インジケータ LED `PinConfig::LED` |

`POT1`、`POT2`、`POT3` はすべて `100k` のポテンショメータです。両端は `+5V` と `GND` に接続され、各ワイパーは Arduino のアナログ入力に接続されます。

`F1` から `F4` は `AudioJack2_SwitchT` 部品として示されています。S 端子は GND に接続されています。T 端子が信号線で、各ジャック信号は抵抗、プルダウン、クランプダイオード、コンデンサを経由して Arduino ピンに到達します。このスケッチでは、`F4` / `D11` をオーディオ出力、`F1` / `A3_D17` を GATE 入力、`F2` / `D9` を KILL 入力、`F3` / `D10` を LFO PAUSE 入力として使用します。

### 2.2 ジャックの入出力設定に関する注意

`F1` から `F4` は Arduino ピンに接続されたジャックなので、ソフトウェア上の `pinMode()` 設定によって入力にも出力にもできます。ただし、各ジャックの周辺回路やピン特性は異なるため、すべての方向に同じように適しているわけではありません。

| Jack | 主な接続ピン | 入出力に関する注意 |
|---|---|---|
| F1 | A3 / D17 | このスケッチでは GATE 入力として使用。外部信号は基板上のプルダウンとクランプを通ります。 |
| F2 | A4 と D9 | このスケッチでは D9 経由の KILL 入力として使用。A4 と D9 が同じ信号線を共有するため、ピン方向に注意が必要です。 |
| F3 | A5 と D10 | このスケッチでは D10 経由の LFO PAUSE 入力として使用。A5 と D10 が同じ信号線を共有するため、ピン方向に注意が必要です。 |
| F4 | D11 | このスケッチではオーディオ出力として使用。デジタル入力としても使えますが、アナログ入力には使えません。 |

`F2` と `F3` で重要なのは、1つのジャック信号線がアナログ入力ピンとデジタル I/O ピンの両方に接続されていることです。たとえば、`F2` は `A4` と `D9` が同じネットを共有し、`F3` は `A5` と `D10` が同じネットを共有します。

このため、外部ジャック信号を `analogRead(A4)` または `analogRead(A5)` で読む場合、対応する `D9` または `D10` を `OUTPUT` に設定しないでください。`D9` または `D10` が出力設定で HIGH または LOW を駆動すると、外部信号やアナログ入力回路と衝突する可能性があります。入力用途では、対応するデジタルピンを `INPUT` のままにして高インピーダンスを保ちます。`INPUT_PULLUP` は、そのバイアスが意図したものの場合だけ使用してください。

逆に、`D9` または `D10` からジャックへ出力する場合、共有されている `A4` または `A5` が入力のままであれば通常は大きな問題になりません。ただし、外部機器も同じジャックを駆動している場合、外部機器と Arduino 出力が衝突する可能性があります。これらのジャックを出力として使う場合は、同じジャックを外部ソースが同時に駆動していない前提にしてください。

Arduino Nano では、`A4` と `A5` は I2C の `SDA` と `SCL` ピンでもあります。将来 I2C デバイスを追加する場合、`F2` / `F3` ジャック回路と I2C 信号が同じピンを共有するため、機能衝突が起きる可能性があります。

`D11` がオーディオ出力に適しているのは、このスケッチが D11 をデジタル出力として HIGH / LOW に駆動して矩形波を生成するためです。デジタル出力ピンに接続されたジャックは、この用途に適しています。`F4` は専用の `D11` ネットとして扱われ、`F2` や `F3` のように別のアナログ入力ピンと共有されていないため、専用オーディオ出力として使いやすい構成です。

ただし、この出力は DAC 音声ではなく 5V 矩形波です。出力音量制御はありません。`F4` 周辺の `R8 1k`、保護ダイオード、`C10 1uF` は、ジャック接続時の保護や高周波ノイズ低減に役立ちますが、通常のラインレベル音声出力そのものではありません。外部ミキサーやモジュラー機器へ接続する場合は、受け側の入力レベルを考慮し、必要に応じて減衰してください。

## 3. 操作仕様

### 3.1 PITCH POT

`A0` の値を読み取り、基本周波数 `baseFreq` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 130Hz-2100Hz

変換には Arduino の `map()` を使います。

```cpp
map(pitchValue, 0, 1023, 130, 2100)
```

### 3.2 AMP POT

`A1` の値を読み取り、LFO の周波数変調幅 `amplitude` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 0-600

この値は Hz として直接扱われ、最終的に `baseFreq + lfoValue` として周波数に加算されます。この値は LED の最大輝度にも使われるため、LFO 深さを大きくすると LED の明るさ変化も大きくなります。

### 3.3 SPEED POT

`A2` の値を読み取り、LFO 位相の増分 `step` に変換します。

- 入力範囲: 0-1023
- 出力範囲: 0.01-0.8

この `step` 値は、更新ごとに `state.angle` へ加算されます。更新間隔は 10ms なので、値が大きいほど LFO 周期が速くなります。

現在の実装では、POT3 は LFO 速度のみを制御します。発音のオン/オフは A3 / D17 GATE 入力、D4 ボタン長押し、D9 KILL 入力で制御されます。D10 LFO PAUSE 入力が HIGH の間は、この速度設定に関係なく LFO 位相が停止します。

### 3.4 GATE 入力

`A3 / D17` はデジタル入力として読み取られます。GATE 入力が HIGH の間だけ D11 へ音を出力します。LOW の間はオーディオエンジンを停止します。音が非アクティブからアクティブへ変化したとき、`state.angle` を `0.0` にリセットし、LFO 位相を先頭から再開します。実際のリセット条件は GATE 状態だけではなく、D9 KILL 入力や D4 ボタン長押しも含めた「非発音状態から発音状態への遷移」です。詳細は「7. GATE 入力と発音動作」を参照してください。

F1 回路には `R12 100k` のプルダウンがあるため、コードではピンを `INPUT` に設定し、`INPUT_PULLUP` は使いません。

### 3.5 ボタン

D4 ボタンの動作は、ボタンを押している時間によって変わります。

- 0.23 秒未満の短押し: 次の LFO 波形へ変更します。
- 0.23 秒以上の長押し: 手動ゲートとして動作し、押している間だけ発音します。

LFO 波形は次の順に切り替わります。

1. `SINE`
2. `SQUARE`
3. `SAWTOOTH`
4. `REVERSE_SAWTOOTH`
5. `SINE` に戻る

短押し判定はボタンを離したときに行われます。発音する長さまでボタンを押していた場合、ボタンを離しても波形は変わりません。

### 3.6 KILL 入力

`D9` はデジタル入力として読み取られます。D9 が HIGH の間は、A3 GATE や D4 ボタン長押しが有効でも、オーディオエンジンを停止して強制ミュートします。KILL 中も LFO と LED インジケータは動き続けます。

### 3.7 LFO PAUSE 入力

`D10` はデジタル入力として読み取られます。D10 が HIGH の間、`state.angle` は進まず、LFO は現在位置に留まります。オーディオピッチと LED 輝度は、一時停止した位置の LFO 値を保持します。

## 4. 周波数生成

通常動作時、周波数は次のように決定されます。

1. ポテンショメータから `baseFreq`、`amplitude`、`step` を読み取ります。
2. D9 KILL 入力が HIGH の場合、オーディオ状態を強制的に非アクティブにします。
3. KILL が LOW で、A3 / D17 GATE 入力または D4 ボタン長押しのどちらかが有効なら、オーディオはアクティブです。両方とも無効なら、オーディオエンジンを停止します。
4. オーディオが非アクティブからアクティブへ変化したら、`state.angle` を `0.0` に設定します。
5. D10 LFO PAUSE 入力が LOW の場合、`step` を `state.angle` に加算します。HIGH の場合は加算しません。
6. `state.angle` が `TWO_PI` 以上の場合、`TWO_PI` を引いて位相を折り返します。
7. 現在の波形に応じて `lfoValue` を計算します。
8. `modulatedFreq` を `baseFreq + lfoValue` に設定します。
9. `modulatedFreq` を 130Hz-3000Hz に制限します。
10. `setAudioFrequency(modulatedFreq)` で D11 の矩形波周波数を更新します。
11. `lfoValue` を 0 から最大輝度までの値へ連続的に変換し、LED PWM 輝度に適用します。最大輝度は LFO 深さに応じます。

## 5. LFO 波形

### 5.1 SINE

```cpp
sin(state.angle) * amplitude
```

サイン波は `-amplitude` から `+amplitude` まで滑らかに変化します。

### 5.2 SQUARE

コードコメントでは `Square-like wave with curved transition sections` と説明されています。実装は完全な矩形波ではなく、固定値の区間とサインカーブ区間を組み合わせています。

- `0 <= angle < PI/2`: `+amplitude`
- `PI/2 <= angle < PI`: `amplitude * sin(...)`
- `PI <= angle < 1.5PI`: `-amplitude`
- `1.5PI <= angle < 2PI`: `-amplitude * sin((angle - 1.5 * PI) * 2)`

一部の区間境界では値が急に変わります。この波形は完全に滑らかではありません。

### 5.3 SAWTOOTH

```cpp
((state.angle / TWO_PI) * 2 - 1) * amplitude
```

この波形は `-amplitude` から `+amplitude` へ直線的に上昇します。

### 5.4 REVERSE SAWTOOTH

```cpp
(1.0 - (state.angle / TWO_PI)) * 2 * amplitude - amplitude
```

この波形は `+amplitude` から `-amplitude` へ直線的に下降します。

状態は `State state` に集約されています。

| メンバー | 役割 |
|---|---|
| `waveformType` | 現在の LFO 波形 |
| `angle` | 現在の LFO 位相 |
| `lastUpdateTime` | 前回の通常動作更新時刻 |
| `buttonPressStart` | ボタンが押された時刻 |
| `buttonWasPressed` | 前回ループでボタンが押されていたか |
| `buttonLongPressActive` | ボタン長押しによる手動発音が有効か |

## 7. GATE 入力と発音動作

現在の実装では、POT3 は無音制御を行いません。発音のオン/オフは A3 / D17 GATE 入力、D4 ボタン長押し、D9 KILL 入力で制御されます。

- D9 KILL HIGH: GATE やボタン状態に関係なく、オーディオエンジンを停止して強制ミュートします。
- D9 KILL LOW かつ GATE HIGH またはボタン長押しが有効: `updateNormalOperation()` を呼び、D11 に矩形波を出力します。
- D9 KILL LOW、GATE LOW、ボタン長押しなし: オーディオエンジンを停止し、D11 を LOW にします。LED はオーディオ状態に関係なく LFO を表示し続けます。

オーディオが非アクティブからアクティブへ変化したとき、スケッチは `state.angle = 0.0` と `state.lastUpdateTime = 0` を設定します。これにより、GATE またはボタン長押しで発音が始まるとき、LFO は先頭から再開します。D10 LFO PAUSE が HIGH の間は、発音中でも LFO 位相は進みません。

## 8. 実装メモ

ポテンショメータ読み取りは、前回の平滑化値との 2 サンプル平均を使います。初期の平滑化値は 0 なので、起動直後の最初の読み取りは実際の値より低くなる場合があります。

`A3 / D17` は、基板側にプルダウンがある前提で `INPUT` に設定されています。`INPUT_PULLUP` は外部 GATE 信号にバイアスを加えるため使用していません。

現在のオーディオエンジンは D11 の矩形波周波数だけを制御するため、このスケッチは音量フェードや真の振幅制御を実装していません。

D3 のハードウェア PWM は ATmega328P の Timer2 を使用するため、ソース内に残している `tone()` フォールバックと衝突する可能性があります。そのため、LED 輝度は `analogWrite()` ではなく、`micros()` を使ったソフトウェア PWM で制御します。LED 輝度は LFO 値から 0 から最大輝度までの範囲へ連続的に変換されるため、A2 が左端付近で LFO が非常に遅い場合でも、ゆっくりした LED の動きが見えます。

## 9. コンパイル互換性メモ

### 9.1 C++17 構造化束縛

以前のバージョンでは C++17 の構造化束縛を使用していました。現在のコードは Arduino AVR 環境との互換性のため、通常の構造体代入を使用しています。

```cpp
PotValues pots = readPotValues();
```

### 9.2 static constexpr float

`AudioConfig` 内の `STEP_MIN` と `STEP_MAX` は `static constexpr float` です。

```cpp
static constexpr float STEP_MIN = 0.01;
static constexpr float STEP_MAX = 0.8;
```

Arduino AVR ターゲットでは、クラススコープの非整数定数は `constexpr` として宣言する方が安全です。

### 9.3 map() の戻り値

Arduino の `map()` は整数演算を使用し、`long` を返します。

このコードでは、`STEP_MIN * 1000` と `STEP_MAX * 1000` を 10-800 にマッピングし、その後 1000.0 で割ることで、小数の step 値を作っています。

## 10. 実装上の注意と改善候補

### 10.1 矩形波出力では音量フェードできない

現在の実装には音量スイープ処理は含まれていません。D11 の矩形波出力は周波数だけを制御し、真の振幅制御を持たないためです。

ポップノイズ低減のために実際のフェードを行うには、PWM 出力、外部 VCA、DAC、またはタイマ割り込みベースの波形生成など、別の方式が必要です。

### 10.2 SQUARE 波形には境界不連続がある

SQUARE 波形はコード内で `Square-like wave with curved transition sections` と説明されていますが、一部の区間境界には急なジャンプがあります。

たとえば、`angle` が `PI/2` に到達した直後、戻り値は `+amplitude` から `0` へジャンプします。急な周波数変化を避けたい場合は、波形設計を見直す余地があります。

### 10.3 最初のポテンショメータ読み取りが低くなる場合がある

平滑化に使う前回値は 0 で初期化されるため、起動直後の最初の読み取りは実際の値の約半分になる場合があります。通常動作中は、前回の平滑化値との 2 サンプル平均として動作します。
